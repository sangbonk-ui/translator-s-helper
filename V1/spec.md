## 1. 프로젝트 개요 ⭐

| 항목 | 내용 |
| --- | --- |
| 프로젝트명 | 식약처 가축 검역 설문서(questionnaire) 번역문 검수  |
| 한 줄 소개 | 에이전시 번역의뢰서 상의 번역 준수 요건을 기준으로, 내가 번역한 번역파일이 해당 요건을 준수했는지 검수하는 웹 앱을 만들고 싶다. |
| 목표 | 대용량 번역물 파일을 일일이 확인할 필요없이 웹 앱을 통해서 해당 용어의 번역이 번역 준수 요건을 충족하는지 파악하여 수정이 용이하도록 표시한다. 웹 앱 사용을 통해서 번역물의 품질 제고를 도모함. |
| 타겟 사용자 | 식약처 가축 검역 설문서 번역가 일반 대상 |
| 플랫폼 | 웹 (모바일 반응형 여부: 불필요) |

## 2. 문제 정의 & 해결 방안

---

- **문제**: 번역물이 대용량인 경우가 종종 있어서(워드 파일 100~200쪽 분량) 확인에 시간이 추가로 소요됨.
- **기존 방식의 한계**: 번역물 파일을 열어서 해당 번역 항목을 찾아 번역 기준 준수 여부를 일일이 확인해야 함.
- **이 앱의 해결 방식**: 원문, 번역문, 번역 준수 기준을 대시보드에 업로드한 후, 수정 버튼을 클릭하면 웹 앱에서 번역문에서 해당 용어의 번역 준수 기준에 대한 충족여부를 확인하고 미충족 항목의 경우 표시하여 수정할 곳을 알려줌.

## 3. 핵심 기능 ⭐

> 
> 

| 우선순위 | 기능명 | 설명 | 비고 |
| --- | --- | --- | --- |
| P0 (필수) | 파일 업로드 | 원문과 번역문의 비교.대조.수정을 원활하게 하기 위한 기능임. | 검사 대상 항목별 검색 |
| P0 (필수) | 자동검수 실행 | 번역후 원문, 번역문 파일을 육안으로 개별적으로 검수.수정하는 작업을 자동으로 수행함. | 원문, 번역문을 화면상에 나란히 배치 |
| P0 (필수) | 검수 결과 표시 | 의뢰처의 번역 준수 기준를 미충족하는 해당 항목을 화면 상에 표시함. | 원문, 번역문의 각 해당 항목 나란히 표시. 형광색 표시 |
| P1 (중요) |  |  |  |
| P2 (나중에) |  |  |  |

### 기능 상세 (P0만이라도)

1. 번역 후 검수 통해서, 번역물이 식약처(에이전시 통해서 입수)의 번역 준수 요건을 준수하였는지 확인하여,
2. 번역물의 미준수 항목에 대한 수정 작업 실시 목적. 일차로 수정 작업 자체는 번역가가 수행함. 수정 작업이 필요한 항목을 확인하고 표시하고, 해당 항목에 대한 수정작업이 수행가능하도록 한다.

### 기능명: 번역 준수 요건 자동 검수

- **사용자 시나리오**: 용자가 원문, 번역문(Word), 준수 기준(Excel/CSV) 파일을 업로드하고 '검수 시작' 버튼을 클릭하면 → 검수 결과가 화면에 표시된다
- **입력**: 버튼 클릭, 텍스트 입력, 파일 업로드 등
- **출력**: 번역문 내 미준수 단어가 형광펜으로 표시된 결과 화면이 나타난다.
- **예외 처리**: "만약 200페이지짜리 파일을 올렸는데 서버가 멈추면 어떻게 할 것인가?", "지원하지 않는 파일 형식(PDF 등)을 올리면 어떤 안내창을 띄울 것인가?”

## 4. 화면 구성 ⭐

> 
> 

### 화면 목록

1. 메인 화면 — 화면을 사분하여 상부 좌측은 원문 배열, 우측은 번역 준수 기준을 미충족한 번역문 배열. 서로 대응이 이루어지도록 배치함. 수정이 필요한 해당 항목을 표시(형광펜으로 표시).
2. (가능하면, 해당 화면에서 수정 작업이 가능하도록) 해당 미충족 번역 항목을 번역문 표시 박스 내에서 수정하는 입력창을 제공.

### 화면별 구성

### [화면 1] 메인

- 상단: 좌우에 원문과 번역문 나란히 배치, 번역문 옆에 검수 수행 버튼 배치
- 중앙: 없음(화면은  상하 기준으로 이등본)
- 하단: 번역 준수 기준 배치
- 이 화면에서 가능한 동작:

### 화면 이동 흐름

```
메인 → (버튼 클릭) → 상세 → (완료) → 결과
```

# **번역 에이전시가 제공하는 번역 준수 기준에 대한 예시**

### <예시1>

1. **영어 지우지 마시고, 영한 병기 형태로 번역 바랍니다.**
2. **원문에서 질문 부분은 영어로 그대로 두도록 한다. 답변 부분만 번역한다.**

## <예시2>

1. **번역문에서. “~한다.”, “~이다.” 등등과 같이 반드시 평서체로 번역한다.**
2. **동물 질병과 관련해서는 surveillance를 예찰로 번역한다.**
3. **‘audit’는 ‘실사’, ‘establishment’는 사전 상의 의미가 ‘시설’인 경우 ‘작업장’으로 번역한다.**

## <예시3>

1. **다음 [용어] 리스트는 고객사에서 수정을 요청한 고유명사(용어) 목록이다.**
2. **[용어]** 원문의 해당 용어를 번역문의 용어로 번역할 것.      ****

| 원문에서 사용된 용어 | 번역문에서 사용된 번역어 | 비고   |
| --- | --- | --- |
| Border Force | 국경검사소 |  |
| ENS | 번역문에서 해당 항목이 1) 안전 및 보안 수입신고 2) 반입신고 3) 수입요약신고 등 여러 용어로 번역되어 있음.  | 용어를 통일할 것. |
| NCTS | 번역문에서 해당 항목이 1) 새로운 전산화 통관시스템 2) 새로운 전산 통관 시스템 3) 전산운송시스템 등 여러 용어로 번역되어 있음. | 용어를 통일할 것. |
| RoRo | 번역문에서 해당 항목이 RoRo로 번역되어 있음.  | 우리말로 번역할 것. |
